# Phase 1 — Computer Fundamentals
## Topic 1.4 — Storage (HDD vs SSD)

---

## 1. TOPIC OVERVIEW

RAM is temporary. So where does everything **permanently live?**

Storage.

Every file, every password database, every log file, every piece of evidence — lives on storage. As a penetration tester you will recover deleted files, analyze logs, and understand how data is stored and erased. This topic explains exactly how that works.

---

## 2. CONCEPT EXPLANATION

Storage is your computer's **long-term memory.**

Unlike RAM, storage keeps data even when power is off.

Two main types:

| Type | Full Name | Technology |
|---|---|---|
| **HDD** | Hard Disk Drive | Magnetic spinning disks |
| **SSD** | Solid State Drive | Flash memory chips (no moving parts) |

> Simple analogy:
> - **HDD** = Old vinyl record player — spinning disk, needle reads data, slower
> - **SSD** = USB flash drive — no moving parts, instant access, faster
> - **RAM** = Whiteboard — fast but wiped clean when done

---

## 3. INTERNAL WORKING

### 🔲 How HDD Works

```
┌─────────────────────────┐
│  Spinning Magnetic Disk │
│    ┌───────────────┐    │
│    │  Track        │    │
│    │   ┌────────┐  │    │
│    │   │ Sector │  │    │
│    │   └────────┘  │    │
│    └───────────────┘    │
│         ↑               │
│    Read/Write Head      │
└─────────────────────────┘
```

- Data is stored as **magnetic patterns** on spinning platters
- A mechanical arm with a **read/write head** moves to find data
- Speed measured in **RPM (Rotations Per Minute)**
- Common speeds: **5400 RPM** (slow) and **7200 RPM** (faster)

**Problem:** Moving parts = slower + can physically break if dropped

---

### 🔲 How SSD Works

- No moving parts
- Data stored in **NAND flash memory chips**
- Electrically reads and writes data instantly
- Same technology as your phone's storage

**Result:** 5x to 10x faster than HDD

---

### 🔲 Speed Comparison

| Type | Read Speed | Write Speed | Noise | Shock Resistance |
|---|---|---|---|---|
| HDD (5400 RPM) | ~100 MB/s | ~100 MB/s | Yes | Low |
| HDD (7200 RPM) | ~150 MB/s | ~150 MB/s | Yes | Low |
| SSD (SATA) | ~500 MB/s | ~450 MB/s | Silent | High |
| SSD (NVMe) | ~3500 MB/s | ~3000 MB/s | Silent | High |

> **NVMe** = Next generation SSD, extremely fast, plugs directly into motherboard

---

### 🔲 How Data is Stored — Bits, Bytes, and File Systems

Everything is stored as **0s and 1s (bits)**

| Unit | Size |
|---|---|
| 1 Bit | Smallest unit (0 or 1) |
| 8 Bits | 1 Byte |
| 1024 Bytes | 1 KB (Kilobyte) |
| 1024 KB | 1 MB (Megabyte) |
| 1024 MB | 1 GB (Gigabyte) |
| 1024 GB | 1 TB (Terabyte) |

**File System** = The method used to organize and store files on storage

| File System | Used By |
|---|---|
| **NTFS** | Windows |
| **FAT32** | USB drives, older systems |
| **ext4** | Linux |
| **APFS** | macOS |

> **Why it matters for hacking:**
> Different file systems store and delete data differently. This affects **data recovery and forensics.**

---

### 🔲 How Deletion Actually Works

This is critical for penetration testers and forensic investigators.

**What most people think:**
> "I deleted the file. It's gone."

**What actually happens:**

```
File deleted
     ↓
OS removes the file's entry from the directory
     ↓
Marks that storage space as "available"
     ↓
Actual data still physically sits on disk
     ↓
Stays there until new data overwrites it
```

The file is **invisible** but **not gone.**

> **Why it matters for hacking:**
> Tools like **Autopsy, PhotoRec, and TestDisk** can recover these "deleted" files. Attackers use this to recover passwords, documents, and evidence. Defenders use **secure wiping** to truly destroy data.

---

### 🔲 Secure Data Deletion

Simply deleting files is not enough. Proper methods:

| Method | How It Works |
|---|---|
| **Overwriting** | Writes random 0s and 1s over old data multiple times |
| **Degaussing** | Strong magnet destroys magnetic data on HDD |
| **Physical Destruction** | Shredding the drive |
| **Encryption then delete** | Encrypt first, then delete — unreadable even if recovered |

**Linux command for secure delete:**
```bash
shred -u -z -n 3 filename.txt
```
This overwrites the file 3 times then deletes it.

---

### 🔲 Partitions

A single physical drive can be divided into **partitions** — separate sections that act like independent drives.

```
Physical Drive (500GB)
├── Partition C: (200GB) — Windows OS
├── Partition D: (200GB) — Files
└── Partition E: (100GB) — Kali Linux
```

> **Why it matters for hacking:**
> Attackers sometimes hide data in **unallocated partitions** that don't appear in normal file browsing.

---

## 4. WHY IT MATTERS

- **Log files** stored on disk contain evidence of attacks
- **Deleted files** can be recovered and contain sensitive information
- **Password databases** (like SAM file in Windows) live on storage
- **Malware persistence** — malware writes itself to storage to survive reboots
- **Forensic investigations** analyze storage to reconstruct what happened

---

## 5. REAL-WORLD EXAMPLE

**Scenario:** A company fires an employee suspected of stealing data.

The employee:
1. Copies confidential files to USB
2. Deletes the copies from their work laptop
3. Empties the Recycle Bin
4. Thinks everything is gone

A forensic investigator:
1. Uses **Autopsy** tool to scan the drive
2. Recovers the "deleted" files
3. Finds the original files + USB transfer logs
4. Recovers timestamps showing exactly when files were copied

**Result:** Evidence recovered. Employee caught.

This exact skill is used in **penetration test reports** when demonstrating that sensitive data is recoverable.

---

## 6. ATTACKER PERSPECTIVE 🔴

| Attack | Storage Involvement |
|---|---|
| **Data theft** | Copying sensitive files from storage |
| **Log tampering** | Deleting or modifying log files on disk |
| **Malware persistence** | Writing malware to startup locations on disk |
| **SAM database theft** | Stealing Windows password file from storage |
| **Deleted file recovery** | Recovering sensitive files victim thought were gone |
| **Steganography** | Hiding data inside innocent-looking files on disk |

---

## 7. DEFENDER PERSPECTIVE 🔵

| Defense | What It Does |
|---|---|
| **Full Disk Encryption (BitLocker/VeraCrypt)** | Makes stolen drives unreadable |
| **Secure file deletion tools** | Actually destroys data instead of just hiding it |
| **Log protection** | Sends logs to remote server so attackers can't delete them |
| **File integrity monitoring** | Alerts when important files are modified |
| **DLP (Data Loss Prevention)** | Detects and blocks unauthorized file copying |

---

## 8. COMMON MISTAKES BEGINNERS MAKE

- Thinking deleted = gone — it never is without secure wiping
- Not encrypting their own hacking lab drives — your practice files could be recovered
- Allocating too little storage to VMs — Kali Linux needs minimum **20GB**, preferably **50GB**
- Confusing **partition** with **physical drive**

## 9. PRACTICAL LAB 🧪

**Objective:** Check your storage details and understand what's on your drive

**On Windows:**
```
1. Press Windows + R
2. Type: diskmgmt.msc
3. Press Enter
4. Observe your partitions, drive sizes, and file systems
```

**On Linux:**
```bash
# Check storage devices and partitions
lsblk

# Check disk usage
df -h

# Check specific disk info
sudo fdisk -l

# See file system types
mount | grep "^/dev"
```

**Expected Output:**
```
NAME   SIZE  TYPE  MOUNTPOINT
sda    500G  disk
├─sda1 100M  part  /boot
├─sda2 450G  part  /
└─sda3 50G   part  /home
```

**Bonus — Recover a deleted file (safe lab practice):**
```bash
# Install TestDisk
sudo apt install testdisk

# Run PhotoRec for file recovery
sudo photorec
```

Try deleting a test file and recovering it — this is basic **digital forensics.**
