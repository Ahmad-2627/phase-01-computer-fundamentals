# Phase 1 — Computer Fundamentals
## Topic 1.1 — What is a Computer? (Hardware Overview)

---

## 1. TOPIC OVERVIEW

Before hacking anything, you need to understand **what you are hacking.**

Every system you will test as a penetration tester runs on hardware. If you don't understand hardware basics, you won't understand why attacks work the way they do.

This topic builds your foundation.

---

## 2. CONCEPT EXPLANATION

A computer is a machine that:
- **Takes input** (keyboard, mouse, network data)
- **Processes it** (does calculations, runs programs)
- **Stores it** (saves files, data)
- **Gives output** (screen, network response)

Simple analogy:

> Think of a computer like a **restaurant.**
> - Kitchen = Processor (does the actual work)
> - Dining tables = RAM (temporary, in-use space)
> - Storage room = Hard Drive (permanent storage)
> - Waiter = Operating System (manages everything)
> - Customers = You / Programs sending requests

---

## 3. INTERNAL WORKING — MAIN HARDWARE COMPONENTS

### 🔲 CPU (Central Processing Unit)
- The **brain** of the computer
- Executes every instruction your programs run
- Speed measured in **GHz** (Gigahertz — how many instructions per second)
- Has **cores** — more cores = more tasks at once

> **Why it matters for hacking:** When you run a password cracking tool, the CPU is doing millions of calculations per second.

---

### 🔲 RAM (Random Access Memory)
- **Temporary** memory — only active while power is on
- Stores data that the CPU is **currently using**
- Measured in **GB** (Gigabytes)

> **Why it matters for hacking:** Malware often hides in RAM. A technique called **memory forensics** helps investigators find it.

---

### 🔲 Storage (HDD / SSD)
- **Permanent** memory — stays even after shutdown
- HDD = Hard Disk Drive (older, slower, magnetic)
- SSD = Solid State Drive (faster, no moving parts)

> **Why it matters for hacking:** Deleted files often still exist on storage. Attackers recover them. Defenders wipe them securely.

---

### 🔲 Motherboard
- The **main circuit board**
- Connects CPU, RAM, Storage, and all other parts
- Everything communicates through it

> Simple analogy: Motherboard = the **roads** connecting all parts of a city.

---

### 🔲 PSU (Power Supply Unit)
- Converts electricity from your wall into power the computer uses
- Not directly related to hacking but important for lab setup

---

### 🔲 GPU (Graphics Processing Unit)
- Designed for graphics but also used for **parallel processing**

> **Why it matters for hacking:** Password cracking tools like **Hashcat** use the GPU because it can run **millions of calculations simultaneously** — much faster than CPU alone.

---

### 🔲 NIC (Network Interface Card)
- Allows the computer to **connect to a network**
- Every NIC has a **MAC Address** — a unique hardware identifier

> **Why it matters for hacking:** Attackers spoof (fake) MAC addresses to hide their identity on a network.

---

### 🔲 BIOS / UEFI Chip
- A small chip on the motherboard
- First thing that runs when you turn on a computer
- We cover this in detail in Topic 1.5

---

## 4. WHY IT MATTERS

As a penetration tester you will:
- Run **Kali Linux** on real or virtual hardware
- Use **RAM-intensive tools** like Metasploit
- Use **GPU-powered tools** like Hashcat
- Investigate how malware uses **hardware resources**
- Understand why a **server behaves differently** than a laptop

Without hardware knowledge, you are just clicking buttons without understanding what's happening underneath.

---

## 5. REAL-WORLD EXAMPLE

**Scenario:** You are hired to test a company's server.

- The server has **32GB RAM** and **16 CPU cores**
- It runs multiple services simultaneously
- Your job is to find weaknesses in those services

If you don't understand that each service is a **process using CPU and RAM**, you won't understand what you're scanning or why certain attacks consume more resources.

---

## 6. ATTACKER PERSPECTIVE 🔴

Attackers think about hardware because:
- **RAM attacks** — injecting malicious code into running processes
- **Cold boot attacks** — stealing encryption keys from RAM after shutdown
- **GPU cracking** — cracking passwords thousands of times faster
- **MAC spoofing** — faking NIC identity to bypass network filters
- **Firmware attacks** — hiding malware inside BIOS/UEFI (very advanced)

> All of this is taught for **defensive understanding.** You learn attacker thinking to protect systems better.

---

## 7. DEFENDER PERSPECTIVE 🔵

Defenders protect hardware by:
- **Encrypting storage** so stolen drives are unreadable
- **Monitoring RAM usage** to detect suspicious processes
- **Locking BIOS** with passwords to prevent tampering
- **Disabling unused ports** (USB, etc.) to prevent physical attacks
- **Using secure boot** to ensure only trusted software loads

---

## 8. COMMON MISTAKES BEGINNERS MAKE

- Thinking hacking is only about software — hardware matters too
- Ignoring RAM limits when setting up a home lab (causes crashes)
- Not understanding why their Kali Linux VM runs slow (not enough RAM allocated)
- Confusing RAM with Storage — they are completely different things

---

## 9. PRACTICAL LAB 🧪

**Objective:** Identify your own computer's hardware specs

**Steps:**

**On Windows:**
```
1. Press Windows + R
2. Type: msinfo32
3. Press Enter
4. Read: Processor, RAM, and other details
```

**On Linux:**
```bash
# Check CPU
lscpu

# Check RAM
free -h

# Check Storage
df -h

# Check all hardware summary
sudo lshw -short
```

**Expected Outcome:**
You will see your CPU model, RAM size, and storage details.

**Key Learning Point:**
You just did your **first system reconnaissance** — gathering information about a system. This is exactly what penetration testers do on target systems.

## 10. KEY TERMS

| Term | Simple Meaning |
|---|---|
| **CPU** | Brain of the computer, runs all instructions |
| **RAM** | Temporary fast memory, lost on shutdown |
| **HDD/SSD** | Permanent storage, survives shutdown |
| **Motherboard** | Main board connecting all components |
| **GPU** | Graphics chip, also used for fast calculations |
| **NIC** | Network card, connects you to a network |
| **MAC Address** | Unique hardware ID of your network card |
| **GHz** | Speed unit of CPU (billions of cycles/second) |
| **Firmware** | Software permanently stored in hardware chips |
| **BIOS/UEFI** | First program that runs when PC starts |
