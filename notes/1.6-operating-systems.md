# Phase 1 — Computer Fundamentals
## Topic 1.6 — Operating Systems (What They Are and Why They Exist)

---

## 1. TOPIC OVERVIEW

BIOS hands control to the **bootloader** which loads the **Operating System.**

Everything you do as a hacker — running tools, exploiting vulnerabilities, managing files — happens through an OS. You need to deeply understand what an OS is, what it controls, and why attackers target it constantly.

---

## 2. CONCEPT EXPLANATION

An **Operating System (OS)** is software that:
- Manages all hardware resources (CPU, RAM, Storage)
- Runs and manages programs
- Controls who can access what
- Acts as a bridge between you and the hardware

Without an OS, you'd need to write machine code directly to talk to hardware — impossible for normal use.

> Simple analogy:
> Think of the OS as a **hotel manager.**
> - Hotel = Computer hardware
> - Guests = Programs/applications
> - Manager = OS
>
> The manager decides:
> - Which guest gets which room (RAM allocation)
> - Who can access which floor (permissions)
> - When rooms get cleaned (memory management)
> - Who gets thrown out (killing processes)

---

## 3. INTERNAL WORKING

### 🔲 Main Components of an OS

```
┌─────────────────────────────────┐
│         User Applications       │  ← Browser, tools, malware
├─────────────────────────────────┤
│         System Calls            │  ← Apps request OS services
├─────────────────────────────────┤
│            Kernel               │  ← Core of OS, full hardware access
├─────────────────────────────────┤
│           Hardware              │  ← CPU, RAM, Storage, NIC
└─────────────────────────────────┘
```

---

### 🔲 The Kernel

The **kernel** is the heart of the OS.

It runs in **Kernel Mode** (remember Topic 1.2) and has complete control over:
- CPU scheduling
- RAM management
- Device communication
- File system access
- Network communication

> **Why it matters for hacking:**
> Getting code to run in the kernel = **full system control.**
> This is why privilege escalation attacks targeting the kernel are so dangerous.

---

### 🔲 System Calls

When a program needs hardware access, it can't do it directly.

It asks the OS using a **system call.**

```
Program says: "I need to read this file"
       ↓
System Call sent to Kernel
       ↓
Kernel checks permissions
       ↓
Kernel reads file from storage
       ↓
Returns data to program
```

> **Why it matters for hacking:**
> Malware analysis involves watching **which system calls malware makes** to understand what it's doing — reading files, connecting to network, creating processes etc.

### 🔲 Why Hackers Use Linux

| Reason | Explanation |
|---|---|
| **Open source** | Full source code visible, fully customizable |
| **Free** | No license cost |
| **Powerful terminal** | Everything controllable via command line |
| **Hacking tools** | Most security tools built for Linux first |

---

### 🔲 Windows vs Linux — Key Differences for Hackers

| Feature | Windows | Linux |
|---|---|---|
| **Market share (desktop)** | ~75% | ~3% |
| **Market share (servers)** | ~25% | ~75% |
| **Hacking tools** | Some | Almost all |
| **Terminal power** | Limited (PowerShell) | Extremely powerful (Bash) |
| **Target for attackers** | Very high (more users) | High (servers) |
| **Cost** | Paid | Free |
| **Customization** | Limited | Unlimited |

---

### 🔲 OS Security Boundaries

The OS enforces **who can do what:**

| Concept | Meaning |
|---|---|
| **Users** | Accounts with specific permissions |
| **Root/Administrator** | Highest privilege account — full control |
| **File permissions** | Who can read, write, execute files |
| **Process isolation** | Programs can't access each other's memory |
| **Sandboxing** | Running programs in isolated environments |

> **Why it matters for hacking:**
> Most attacks try to go from a **normal user** to **root/administrator** — this is **privilege escalation.**

---

### 🔲 OS Vulnerabilities

Operating systems have bugs. Bugs become vulnerabilities. Vulnerabilities become exploits.

Famous examples:

| Vulnerability | OS | Impact |
|---|---|---|
| **EternalBlue (MS17-010)** | Windows | Used in WannaCry ransomware, affected millions |
| **Dirty COW (CVE-2016-5195)** | Linux | Privilege escalation on almost all Linux systems |
| **PrintNightmare** | Windows | Remote code execution via print spooler |
| **Shellshock** | Linux/Mac | Remote code execution via Bash |

> These are studied for **defensive and educational purposes.**

## 4. WHY IT MATTERS

- You will attack and defend **Windows and Linux systems**
- Understanding the kernel helps you understand **privilege escalation**
- System calls reveal **what malware is doing**
- OS vulnerabilities are the most common attack surface in penetration testing
- Knowing both OS types makes you a **complete penetration tester**

---

## 5. REAL-WORLD EXAMPLE

**Scenario — WannaCry Ransomware (2017)**

- Exploited **EternalBlue** — a Windows OS vulnerability
- Spread automatically across networks
- Encrypted files on infected machines
- Demanded Bitcoin ransom
- Affected 200,000+ computers in 150 countries
- Shut down hospitals, banks, and government systems

**Root cause:** Unpatched Windows OS vulnerability.

**Lesson:** Keeping OS updated is one of the most important security practices.

---

## 6. ATTACKER PERSPECTIVE 🔴

| Attack | OS Involvement |
|---|---|
| **Kernel exploits** | Attack kernel to gain full system control |
| **Privilege escalation** | Move from normal user to root/admin |
| **OS vulnerability exploitation** | Use unpatched bugs like EternalBlue |
| **Malware persistence** | Write to OS startup locations |
| **DLL hijacking** | Trick Windows OS into loading malicious files |
| **Living off the land** | Use built-in OS tools for attacks (PowerShell, WMI) |

---

## 7. DEFENDER PERSPECTIVE 🔵

| Defense | What It Does |
|---|---|
| **OS patching** | Fix known vulnerabilities regularly |
| **Least privilege** | Give users minimum permissions needed |
| **Endpoint Detection (EDR)** | Monitor OS for suspicious behavior |
| **Application whitelisting** | Only allow approved programs to run |
| **OS hardening** | Disable unnecessary services and features |
| **Logging** | Record OS events for investigation |

---

## 8. COMMON MISTAKES BEGINNERS MAKE

- Thinking Linux is only for experts — it's learnable step by step
- Running everything as **root on Kali** — dangerous habit, even in labs
- Not updating their OS — leaves known vulnerabilities open
- Ignoring Windows internals — most corporate targets run Windows
- Thinking one OS is enough — real pentesters must know both

## 9. PRACTICAL LAB 🧪

**Objective:** Explore your current OS details and compare with Linux

**On Windows:**
```powershell
# Open PowerShell and run:

# OS version
winver

# Detailed system info
systeminfo

# Running processes
tasklist

# Current user
whoami

# User privileges
whoami /priv
```

**On Linux (if already installed):**
```bash
# OS version
uname -a

# Detailed OS info
cat /etc/os-release

# Running processes
ps aux

# Current user
whoami

# User privileges
id
```

**What to observe:**
- Your OS version and build number
- How many processes are running right now
- What user you are logged in as
- What privileges your account has

**Key Learning Point:**
`whoami` and `id` are **the first commands penetration testers run** after gaining access to a system — to know who they are and what they can do.
